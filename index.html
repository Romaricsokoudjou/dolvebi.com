Technical Documentation

Dolvebi — Architecture Guide

Recommended Tech Stack

Frontend: Next.js 14 (App Router) + TypeScript + Tailwind CSS + Framer Motion

Backend: Node.js + Express.js + REST API (or tRPC)

Database: PostgreSQL (via Supabase) + Redis (caching)

Auth: NextAuth.js + JWT + bcrypt

Storage: Cloudinary (product images)

Payments: Stripe API + CinetPay (Africa) + MTN MoMo API

Deploy: Vercel (Frontend) + Railway or Render (Backend)

Database Schema
users { id, email, password_hash, first_name, last_name, 
        phone, country, role (buyer|seller|admin), 
        wallet_balance, created_at, verified }

products { id, seller_id, name, description, price_fcfa, 
           category, images[], stock, status, created_at }

orders { id, buyer_id, seller_id, product_id, quantity, 
         total_fcfa, commission_fcfa (5%), status, 
         shipping_address, created_at }

transactions { id, user_id, type (credit|debit), 
               amount, description, reference, 
               created_at }

wallets { id, user_id, balance, currency (FCFA), 
          total_earned, total_withdrawn }

listings_realestate { id, title, type (D2|D3), 
                       location, price, rooms, 
                       area_m2, images[], status }

courses { id, title, description, price, duration, 
          level, enrolled_count, instructor }

Commission Logic

On each successful order:

const commission = orderTotal * 0.05;        // 5%
const sellerEarning = orderTotal - commission; 
await creditWallet(sellerId, sellerEarning);
await creditWallet(adminId, commission);
